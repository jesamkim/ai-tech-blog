## 구조 (7개 섹션)

### 1. 왜 비디오 RAG인가 — 텍스트 RAG를 넘어서
- 텍스트 RAG는 이미 성숙 (chunking → embedding → retrieval → generation)
- 하지만 기업 데이터의 상당 부분이 비디오 (CCTV, 회의 녹화, 교육, 마케팅)
- Cisco 예측: 2025년 기준 인터넷 트래픽의 82%가 비디오
- VideoRAG (Jeong et al., 2025, ACL Findings, arXiv:2501.05874) 논문이 비디오 RAG 프레임워크를 제안한 배경
- 기존 접근의 한계: 텍스트 변환 시 멀티모달 정보 손실, 쿼리 기반 검색 없이 사전 정의된 비디오 사용

### 2. 실전 유즈케이스 — 비디오 RAG가 풀 수 있는 문제들
- **산업 현장 안전 관리 (건설/제조)**
  - CCTV 아카이브에서 사고 사례 검색, 안전 교육 자료 추출
  - 실시간 감지(안전모 미착용 등)는 스트리밍 추론 — RAG과 별개 아키텍처임을 명시
  - "지난달 크레인 근접 작업 위반 영상 찾아줘" → Video RAG 적합
- **미디어/방송 아카이브**
  - 수만 시간 방송 아카이브에서 특정 장면/인물/발언 검색
  - 텍스트+비주얼 멀티모달 검색의 시너지
- **교육/이러닝**
  - 강의 영상 라이브러리에서 개념별 검색
  - 학습자 맞춤 콘텐츠 큐레이션
- 각 유즈케이스별 실시간 vs 사후 검색(RAG) 경계 짚기

### 3. 비디오를 벡터로 만드는 두 가지 접근법
- **접근법 A: 프레임 임베딩 (Frame-level Embedding)**
  - 비디오에서 N개 프레임 추출 → 각 프레임을 이미지 임베딩 모델로 변환 → 평균
  - 장점: 단순, 기존 이미지 임베딩 인프라 재사용
  - 단점: 시간적 맥락 소실, 오디오/대화 무시, 프레임 간 의미 희석
- **접근법 B: VLM(Video Language Model) 기반 파이프라인**
  - 비디오 전체를 VLM에 입력 → 텍스트 설명 생성 → 텍스트 임베딩
  - 장점: 시간적 맥락/관계/이벤트 포착, 오디오/대화 반영 가능
  - 단점: 추가 추론 비용, 파이프라인 복잡성
- 이론적 분석: 왜 프레임 평균이 정보를 잃는지 (벡터 평균의 의미적 중심점 이동)

### 4. Twelve Labs 모델 — Pegasus v1.2와 Marengo Embed 3.0
- **Twelve Labs 소개**: 비디오 이해 전문 AI 스타트업. SageMaker HyperPod에서 학습.
- **Pegasus v1.2 (비디오 언어 모델)**
  - 아키텍처: video encoder + video-language alignment + language decoder (약 80B 파라미터 - Pegasus-1 기준, Technical Report arXiv:2404.14687, Jung et al.)
  - 최대 1시간 비디오 지원, 15분까지 일정한 TTFT 레이턴시
  - VideoMME-Long 벤치마크에서 GPT-4o, Gemini 1.5 Pro 대비 SOTA 성능 (공식 블로그 기준)
  - 타임스탬프 기반 시간 이해 탁월 (미식축구 예시)
  - Bedrock Inference Profile: us.twelvelabs.pegasus-1-2-v1:0
  - 리전: US West (Oregon), Europe (Ireland) — cross-region inference
- **Marengo Embed 3.0 (멀티모달 임베딩)**
  - 512차원 벡터 출력
  - 입력: 텍스트 + 이미지 (Bedrock에서는 비디오 직접 임베딩 미지원!)
  - Bedrock Inference Profile: us.twelvelabs.marengo-embed-3-0-v1:0
  - 리전: US East (N. Virginia)
  - Marengo 2.7은 비디오 직접 임베딩 지원 (StartAsyncInvoke API)
- **Bedrock 가격** (공식 pricing 페이지 기준):
  - Pegasus: $0.00049/초 (비디오 입력) + $0.0075/1K 토큰 (출력)
  - Marengo Embed: 텍스트/이미지는 동기 InvokeModel, 비디오는 비동기 StartAsyncInvoke

### 5. 실전 실험: Bedrock에서 직접 비교해 보니
- **실험 환경**: Jay의 PoC — GitHub 레포 https://github.com/jesamkim/bedrock-twelvelabs
- **샘플 비디오 3종** (Pexels CC0):
  - nature.mp4: 카파도키아 열기구 (19.3s)
  - city.mp4: 강변 대화 (37.9s)
  - cooking.mp4: 실험실 피펫 작업 (14.0s) — 파일명과 실제 내용 불일치 (Pegasus가 정확히 파악!)
- **실험 1: Pegasus 비디오 QA** (01_pegasus_video_qa.py)
  - 3 비디오 × 4 프롬프트 = 12건 모두 성공 (100%)
  - 한국어 응답: 자연스러운 한국어 생성, 다만 일부 hallucination 존재
  - 타임스탬프 추출: [MM:SS-MM:SS] 형식으로 구간별 설명
  - cooking.mp4의 실제 내용(실험실 피펫)을 정확히 파악 (파일명에 속지 않음)
- **실험 2: 임베딩 비교** (02_marengo_embed.py)
  - Frame-avg: 비디오에서 4프레임 추출 → 이미지 임베딩 평균
  - Pegasus+Marengo: Pegasus 설명 텍스트 → Marengo 텍스트 임베딩
  - **결과 데이터** (정확한 수치!):
    | 쿼리 | 방식 | nature | city | cooking |
    |-------|------|--------|------|---------|
    | "hot air balloons in cappadocia" | Frame-avg | 0.043 | -0.051 | -0.058 |
    | | Pegasus+Marengo | 0.421 | 0.176 | 0.117 |
    | "people having conversation outdoors" | Frame-avg | 0.006 | 0.069 | -0.028 |
    | | Pegasus+Marengo | 0.290 | 0.616 | 0.158 |
    | "science experiment in lab" | Frame-avg | -0.032 | -0.043 | 0.068 |
    | | Pegasus+Marengo | 0.222 | 0.254 | 0.562 |
  - Frame-avg 최고 스코어: 0.069, Pegasus+Marengo 최고: 0.616
  - 두 방식 모두 정확한 1위 매칭 (3/3), 하지만 마진이 극적으로 다름
  - Frame-avg: 프레임 간 cosine similarity 0.74~0.79 (모든 비디오가 비슷해 보임)

### 6. 비디오 RAG 파이프라인 아키텍처 — 권장 패턴
- **인덱싱 시점**: Video(S3) → Pegasus 설명 생성 → Marengo 텍스트 임베딩 → Vector DB 저장
- **검색 시점**: Query → Marengo 쿼리 임베딩 → Vector DB cosine search → Top-K 비디오 → Pegasus 상세 QA
- AWS 서비스 구성: S3 (비디오 저장) + Bedrock (Pegasus + Marengo) + OpenSearch/Pinecone (벡터 DB)
- S3 Vectors 언급 (AWS 케이스 스터디에서 TwelveLabs + S3 Vectors 통합 소개)
- 메타데이터 활용: video_key, description, timestamps를 벡터와 함께 저장
- 스케일링 고려사항: 비동기 인덱싱 (수천 개 비디오), 캐시 전략

### 7. 비용, 한계, 그리고 전망
- **비용 분석**: 10초 비디오 Pegasus 요약 = $0.00049×10 + 출력 토큰 ≈ $0.02
- **한계점**:
  - Marengo Embed 3.0은 Bedrock에서 비디오 직접 임베딩 미지원 (image/text만)
  - Pegasus가 일부 hallucination 발생 (한국어 요약 시 "페가수스 공원" 같은 환각)
  - 리전 제한: Pegasus는 US/EU만, Marengo 3.0은 us-east-1만
- **Marengo 2.7 비디오 임베딩**: 비동기 API로 직접 비디오 임베딩 가능 (향후 Embed 3.0도 지원 확대 예상)
- **전망**: 비디오 RAG는 아직 초기이지만, 텍스트 RAG가 걸어온 길을 빠르게 따라올 것
- VAST Data + TwelveLabs 파트너십 (on-premise 비디오 인텔리전스)
- 비디오가 텍스트만큼 검색 가능해지는 미래
